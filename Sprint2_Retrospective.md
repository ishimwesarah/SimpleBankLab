# Sprint 2 Retrospective - SimpleBank


## What Went Well

1. The team applied lessons from Sprint 1 by improving validation and error handling.

2. New features were delivered through small incremental commits instead of making large changes at once.

3. Additional unit tests helped verify that withdrawal functionality worked correctly.

4. GitHub Actions continued providing automatic validation after code changes were pushed.


---

# What Could Be Improved

1. The application currently stores data in memory, meaning account information is lost when the program closes.




---

# Actions for Future Sprints

1. Introduce persistent storage using a database or file storage.

2. Increase test coverage for all banking operations.




---

# Lessons Learned

- Agile development helps deliver working features step by step instead of waiting until the end.

- Retrospectives help identify improvements that can be applied in future iterations.

- Continuous integration helps detect problems early and improves software quality.
