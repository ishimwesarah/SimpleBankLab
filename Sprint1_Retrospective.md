# Sprint 1 Retrospective - SimpleBank


## What Went Well

1. Development was completed incrementally using small Git commits.

2. Unit testing helped verify that account creation and deposit functionality worked correctly.

3. The GitHub Actions pipeline provided automatic validation after changes were pushed.


## Challenges

1. Setting up JUnit manually required additional configuration because the project does not use Maven.

2. The project structure needed careful management to avoid compilation issues.


## Improvements for Sprint 2

1. Improve input validation to handle incorrect user input more safely.

2. Add more banking operations such as withdrawal and balance checking.

3. Improve monitoring by adding more meaningful application logs.
